# FitConnect - Project Instructions

## Overview

## Key Features Implemented
1. **Dynamic plan limits** - Professor dashboard and alunos-manager use dynamic maxStudents from stats API (not hardcoded)
2. **Invite flow** - Professor invites student by email → PendingInvite record created → Email sent with `/signup?type=aluno&email=xxx` link → On signup, auto-links via PendingInvite table
3. **Professor invite code** - 6-char alphanumeric code (auto-generated), shown in alunos page with copy button. Students can enter code during signup to auto-link.
4. **Pending invites display** - Alunos page shows pending invites (aguardando cadastro) with cancel option
5. **Student personal workouts** - Students can create their own workouts (isPersonal=true, professorId=null). Aluno treinos page has tabs: "Do Professor" / "Pessoais"
6. **Search/filters** - All list pages (alunos, treinos professor, treinos aluno) have search bars
7. **Workout timer** - ExecutarTreino has elapsed timer and cancel button
8. **Mobile UX** - min-h-[44px] on buttons/inputs for touch targets

## Schema Notes
- Professor.inviteCode: String? @unique - 6-char code
- PendingInvite model: email + professorId (unique combo)
- AssignedWorkout.professorId: String? (nullable for personal workouts)
