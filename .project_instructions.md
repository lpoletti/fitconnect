# FitConnect - Project Instructions

## Overview
FitConnect is a fitness platform connecting personal trainers (professors) and students (alunos). Built on Abacus AI platform.

## Deployment Flow
- **Staging**: `stagfitconnect.abacusai.app` (tag: `staging`) - deploy here first
- **Production**: `fitconnect.abacusai.app` (tag: `prod`) - only after user validation

## Auth
- Professor: john@doe.com / johndoe123
- Student: aluno@teste.com / aluno123
- Google SSO configured (Client ID: 417139493593-...)
- JWT callback always fetches from DB to get professorId/studentId

## NavItems Pattern
Each component defines navItems locally:
- **Professor**: Dashboard, Alunos, Treinos, Meu Plano
- **Aluno**: Dashboard, Meus Treinos, Avaliações, Calendário, Histórico

## Key Features
1. **Multi-media per exercise** (up to 3 files via mediaFiles Json field)
2. **Google SSO** login
3. **Media Lightbox** component for viewing media
4. **Workout propagation** - template updates propagate to assigned student workouts (matches by workoutName + professorId)
5. **Student Evaluation** - 3-section form (Dados+Anamnese, Medidas+Fotos, Objetivos+Termo)
   - Professor creates evaluations: `/professor/alunos/[id]/avaliacao`
   - Student views evaluations: `/aluno/avaliacoes`
   - Photos uploaded via S3 presigned URLs
   - Schema: `StudentEvaluation` model with ~30 fields

## S3 Upload Pattern
Uses `/api/upload/presigned` → PUT to signed URL → build public URL from NEXT_PUBLIC_AWS_BUCKET_NAME/NEXT_PUBLIC_AWS_REGION env vars

## Components Location
- `components/shared/dashboard-shell.tsx` - DashboardShell layout
- `components/professor/` - all professor components
- `components/aluno/` - all student components
- `components/ui/` - shadcn UI primitives

## Design
- Dark/light theme via next-themes
- Font: display font for headings
- Cards with `shadow-[var(--shadow-md)]` and `bg-card rounded-xl`
- Status badges: emerald for active, amber for pending, destructive for issues
